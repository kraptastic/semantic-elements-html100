on semantic elements

Elements designated as semantic use HTML tags that give an indication of the element's meaning to browsers and developers. Before this practice, pages had many indistinguishable elements that required IDs and classes for selection and identification. Computers and international users did not always benefit from the programmer's choice of ID and class names. While some semantic elements outline the document (<header>, <nav>, <section>) there are also elements with specialized functionality such as <datalist> and <figure>. Each of these elements has accessiblity and searching as main considerations.

Consider the use of <section> in comparison to a typical <div> with a class or ID- a <section> is going to behave as a typical <div>, but it's name clearly displays that it is a "section" of the page's outline. Further reading shows the <section> element may contain individual article elements and should have a heading. This standard saves keystrokes and headaches for developers and people or machines accessing the web content. Another semantic element is <nav>, used to indicate links for navigating around the page(s). Interestingly, there is also an element called <aside>, which is usually not viewed by assistive technologies. I am excited to get to use <figure> tag which seems to be a helpful format to enclose <img> elements with accessible information, yet leaving the information readily available for CSS formatting.

An aside - W3schools said that <asides> don't always get targeted by accessiblity software.
